Tensor

Tensor is a polling monitoring agent and metric router for Riemann. Tensor is asynchronous and able to handle polling thousands of metrics at high frequency, receiving them through other protocols, and reliably delivering them to Riemann.

Tensor uses a plugin architecture for input sources and outputs, and can be easily extended using Python to monitor anything or produce any metrics.

Usage

Tensor is easy to install and configure, and supports state thresholds with regex matching for sources which provide multiple metrics. ::

ttl: 60.0
interval: 1.0

sources:
    - service: load
      source: tensor.sources.linux.basic.LoadAverage
      interval: 2.0

    - service: cpu
      source: tensor.sources.linux.basic.CPU
      interval: 2.0
      critical: {
        "^cpu$": "> 0.1"
      }

    - service: memory
      source: tensor.sources.linux.basic.Memory
      interval: 2.0

outputs:
    - output: tensor.outputs.riemann.RiemannTCP
      server: riemann.acme.com
      port: 5554
      tls: true
      cert: /var/lib/puppet/ssl/certs/test.acme.com.pem
      key: /var/lib/puppet/ssl/private_keys/test.acme.com.pem

Consult the documentation for installation instructions and more configuration examples.
